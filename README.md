🎓 C语言协作项目：学生综合教务管理系统

📅 项目概况

本项目旨在通过团队协作，开发一个功能完备的学生选课与成绩管理系统。项目分为两个阶段：

阶段一：构建基础的成绩管理内核，实现数据的 CRUD 与统计分析。

阶段二：基于内核进行架构升级，引入指针操作、注册登录、课程维护及选课系统。

________________________________

🚀 阶段一：基础架构与成绩管理 (Foundation)

👥 团队分工 (6人组)

🧑‍💻 学生 A：项目架构师 & 主菜单

职责：

设计程序整体架构和函数接口。

实现主菜单循环和程序流程控制。

协调各模块间的数据传递。

🧑‍💻 学生 B：数据输入与验证

职责：

设计学生信息输入界面。

实现成绩数据输入功能。

添加输入数据验证（0-100分）。

防止非法字符输入导致系统崩溃。

🧑‍💻 学生 C & D：计算核心模块

职责：

实现总分计算函数。

实现平均分计算函数。

设计成绩统计功能。

处理浮点数计算精度问题。

🧑‍💻 学生 E：等级评定模块

职责：

设计等级评定标准（如 A/B/C/D/F）。

实现等级判断函数。

添加自动评语生成功能。

设计等级分布统计。

🧑‍💻 学生 F：查询与显示模块

职责：

设计成绩查询功能（按学号/姓名）。

实现结果格式化显示（对齐、制表）。

创建成绩报告单。

设计数据展示界面。

⚙️ 技术指标

基础功能：支持至少 5名学生，每名学生 3门课程；具备完整的增删改查（CRUD）。

扩展挑战（可选）：添加图形界面（如使用 EGE 库）。

考核标准：

功能完整性 (30%)：所有基础功能实现。

代码质量 (25%)：规范、注释、可读性。

团队协作 (20%)：沟通、配合、Git提交/代码合并。

系统稳定性 (15%)：错误处理、边界测试。

用户体验 (10%)：界面友好、交互逻辑顺畅。

🛠️ 阶段二：系统升级与选课追踪 (Advanced)

📋 升级概述

基于阶段一的基础，系统需升级为支持 “用户鉴权 + 课程维护 + 选课系统” 的综合平台。重点考核 指针操作、结构体数组 及 模块化编程 能力。

👥 升级版分工细则

🧑‍💻 学生 A：系统控制中心

主函数重构：修改主函数以适配新功能调用，整合注册与登录入口。

流程控制：实现多级循环菜单，确保系统可重复操作。

数据中枢：

声明全局/局部数据结构（基础数据定义）。

关键技术：通过指针将数据结构传递给各功能子模块。

🧑‍💻 学生 B：学生信息管理 (升级版)

核心功能：实现学号、姓名的录入与去重，确保信息唯一性。

具体要求：

输入规范：提示输入学号（非空、长度<19）、姓名（非空、长度<19）。

学号去重：遍历已录入学生，通过指针操作数组校验新学号是否存在。

容量控制：设置学生上限，若达上限提示“学生数已满”。

🧑‍💻 学生 C & D：课程信息维护 (新模块)

核心功能：课程的增删改查，数据合法性校验。

具体要求：

子菜单设计：“1. 录入课程 2. 修改课程 3. 删除课程”。

录入课程：参考现有课程录入至少5门。校验编号唯一、学分合法（0.5-6.0）、名称非空。

修改课程：通过课程编号查找（指针遍历），允许修改名称和学分。

删除课程：查找并删除课程，关键技术：实现数组移位操作，将后续课程数据向前移动。

🧑‍💻 学生 E & F：选课操作模块 (新模块)

核心功能：处理学生选课与退课逻辑，维护学生-课程关系。

具体要求：

子菜单设计：“1. 选课 2. 退课”。

选课流程：

输入学号与课程编号，通过指针查找对应索引。

校验逻辑：学生未选该课、选课数未达上限、课程存在。

数据存储：校验通过后，将课程索引存入学生选课列表（建议使用二维数组或结构体链表）。

退课流程：

校验学生是否已选该课程。

校验通过后，从选课列表中移除该课程。
